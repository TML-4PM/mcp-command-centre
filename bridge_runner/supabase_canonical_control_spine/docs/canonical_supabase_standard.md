# Tech 4 Humanity Canonical Supabase Standard

## Single Source of Truth for Data, Runtime, Evidence, and Control

**Document status:** Canonical  
**Authority level:** Primary  
**Applies to:** All current and future T4H systems, products, agents, dashboards, publishers, and command-centre surfaces  
**Primary platform:** Supabase  

## Purpose

Supabase is the canonical control, state, evidence, and governance spine for the Tech 4 Humanity operating system.

It is the system of record for:
- runtime state
- evidence and proof
- orchestration metadata
- product and business registries
- UI widget definitions
- closure status
- telemetry
- commercial records
- shared operational context
- reality classification

Where another tool performs execution, publishing, storage, messaging, or frontend display, Supabase remains the authoritative source of status, truth, and recoverable state unless explicitly marked otherwise.

## Core principle

Supabase is the canonical source of truth for:
- what exists
- what ran
- what succeeded
- what failed
- what is real
- what is pending
- what is blocked
- what is visible in the command centre
- what is commercially active
- what is evidence-backed

Supabase is not automatically the canonical store for:
- large binary files
- raw media assets
- source code repos
- long-form documents
- third-party app-native content

Those may live elsewhere, but their registry, status, evidence pointer, and lifecycle state should still be recorded in Supabase.

## One-line definition

> Supabase is the canonical control, state, evidence, and governance spine for the Tech 4 Humanity operating system.
